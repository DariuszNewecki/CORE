## What surfaced

### Real governance debt

- `src/api/main.py:18` — `from body.infrastructure.lifespan import core_lifespan`. API layer importing Body directly. The file's docstring claimed "Resolves architecture.api.no_body_bypass violation" while still bypassing — the claim held only because the narrow forbidden list and the fnmatch blind spot together made the bypass invisible. Needs a remediation decision: exclude `src/api/main.py` with rationale (lifespan wiring is composition-root necessity) vs. move `core_lifespan` out of Body (to Will or to `api/` itself).

### Systemic engine bug

- **fnmatch include/exclude asymmetry in `AuditorContext.get_files`** (`src/mind/logic/engines/ast_gate/base.py` or wherever the context file resolver lives — line 48748 in the export). The **exclude** path hand-rolls `**`-aware pattern matching correctly. The **include** path uses raw `fnmatch.fnmatch`, which does not treat `**` as "zero or more directories." Result: every rule scoped `src/X/**/*.py` silently misses `src/X/*.py` top-level files.
- Empirical: `fnmatch.fnmatch('src/api/main.py', 'src/api/**/*.py')` → `False`. Same for `pathlib.PurePath.match`.
- Impact across codebase inventoried in the nested-scope audit: 4 rules with real hidden debt, 3 of which tactically patched today; 1 (`logic.di.no_global_session`) still exposed and expected to surface ~14 findings when the engine is fixed.

### Rule that isn't firing despite failing classes

- `autonomy.tracing.mandatory` is mapped, in-scope, and `SelfHealingAgent` empirically has zero `self.tracer.record` calls in its body. Rule should produce a WARNING finding. Produces zero. Either the rule isn't executing, findings are being filtered before display, or `generic_primitive + required_calls` has a dispatch bug. Not diagnosed this session — parked for dedicated investigation.

### Engine-level gaps identified

Three rules' annotated "gaps" turned out to be engine-level, not mapping-level — none fit B.i shape:
